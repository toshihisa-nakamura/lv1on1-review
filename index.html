Bコーチ
コーチ 日程入力画面
URLを開いて名前を選び、対応できる日付と開始時間を入力。終了時間（+75分）は自動計算。一時保存→完了→送信の3ステップ。
確認してほしいポイント
名前プルダウンは使いやすいか
日付・時間の入力はスムーズか
行の追加・削除は分かりやすいか
保存→完了→送信の流れは自然か

クライアント 予約画面
受講番号・氏名・メールアドレスを入力してから、希望の日程を1つ選んで予約。コーチ名は表示されません。予約後は確認メールが自動送信されます。

業務フロー図
📊 ゼネラルフロー（全体の流れ）
スタートから完了までの全ステップ。③コーチ入力・⑥クライアント予約はサブフロー（二重枠）として参照。
横スクロールで全体を確認できます
🟢 コーチUIフロー（③の詳細）
URLを開く → 前回入力の復元 → 入力・編集 → 保存→完了→送信 → 変更・再送信のフロー。
🟠 クライアントUIフロー（⑥の詳細）
情報入力 → 枠選択 → 確定（仮ロック）→ 送信（予約）→ 取消・変更ループ → 締め切り後：確定。
